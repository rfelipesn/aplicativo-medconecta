3. Build passa mas deploy falha → ESTE é o problema real, e eu estava cego pra ele porque só lia buildLogs.

Os logs de runtime mostram: vazio + Stopping Container. O nginx nem chega a servir. Causa raiz:

O nginx está fixado em listen 8080, mas o Railway injeta uma variável $PORT própria (dinâmica) e faz o health check nessa porta. Como o nginx não escuta na porta que o Railway espera, o health check falha e o Railway mata o container → Stopping Container → FAILED.

Piora: o startCommand no Railway está como nginx -g 'daemon off;', o que ignora o entrypoint oficial da imagem nginx (/docker-entrypoint.sh) — justamente o que faria a substituição de variáveis. Ou seja, mesmo se eu usasse template, ele seria pulado.

É por isso que web E mobile falham idênticos e rápidos, enquanto a API (Node) sobe normal — a API lê process.env.PORT, o nginx não.

Plano de correção definitivo (1 causa → 1 fix)
Ideia central: fazer o nginx escutar em $PORT via o mecanismo de template oficial da imagem nginx, com filtro de envsubst (pra não estragar variáveis internas do nginx como $uri), e devolver o startCommand ao entrypoint padrão.

A. apps/web/nginx.conf → vira template apps/web/default.conf.template
server {
    listen ${PORT};
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    location = /health { access_log off; return 200 "OK\n"; add_header Content-Type text/plain; }

    location / { try_files $uri $uri/ /index.html; }

    location ~* \.(js|css|png|jpg|jpeg|gif|svg|woff2?|ttf)$ {
        expires 30d; add_header Cache-Control "public, immutable";
    }
}
(idêntico para apps/mobile/default.conf.template)

B. apps/web/Dockerfile — stage final reescrito
# ---- Build stage ----
FROM node:20 AS build
WORKDIR /app
ENV NODE_ENV=production
COPY package.json package-lock.json tsconfig.base.json ./
COPY packages/shared ./packages/shared
COPY apps/web ./apps/web
RUN npm install --include=dev --no-audit
ARG VITE_API_URL
ARG VITE_SUPABASE_URL
ARG VITE_SUPABASE_ANON_KEY
ENV VITE_API_URL=$VITE_API_URL VITE_SUPABASE_URL=$VITE_SUPABASE_URL VITE_SUPABASE_ANON_KEY=$VITE_SUPABASE_ANON_KEY
RUN npm run build -w @medconecta/shared
RUN npm run build -w @medconecta/web

# ---- Production ----
FROM nginx:alpine
ENV PORT=8080
ENV NGINX_ENVSUBST_FILTER=PORT           # só substitui ${PORT}, preserva $uri etc.
COPY --from=build /app/apps/web/dist /usr/share/nginx/html
COPY apps/web/default.conf.template /etc/nginx/templates/default.conf.template
EXPOSE 8080
# SEM CMD/startCommand custom → usa o entrypoint oficial que roda o envsubst e sobe o nginx
