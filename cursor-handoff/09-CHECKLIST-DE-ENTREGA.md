# Checklist de entrega para o Cursor

Antes de considerar qualquer tarefa concluída:

- [ ] Leu a regra persistente e o handoff.
- [ ] Conferiu o Git antes de editar.
- [ ] Preservou mudanças existentes e arquivos fora do escopo.
- [ ] Confirmou o fluxo real, não apenas a tela.
- [ ] Manteve contratos shared, API e clientes alinhados.
- [ ] Validou autorização/RLS quando há dados clínicos.
- [ ] Não expôs secrets ou PHI.
- [ ] Executou typecheck/lint/build/testes pertinentes.
- [ ] Fez validação manual quando automação não cobre o fluxo.
- [ ] Informou exatamente os arquivos alterados.
- [ ] Registrou resultados, warnings, riscos e próximos passos.
- [ ] Não fez commit, push, migration ou deploy sem autorização.

## Definição de pronto do handoff

Uma nova sessão deve conseguir explicar o produto, localizar os módulos, preservar o worktree, iniciar os serviços, identificar a tarefa prioritária e continuar sem depender de conversas anteriores.
